# Centerのエージェントプールの使い方

**1. 前提条件**

エージェントプール機能を利用する前に、以下を確認してください：

- akaBot Centerがインストールされアクセス可能であること。
- PostmanなどのHTTPクライアントが利用可能であること。
- HTTPリクエストでエージェントプールをトリガーする場合、Managed Token Guidelineに従って管理トークンを作成する必要があります。

**2. エージェントプールの作成**

**Agent Pool**は特定のワークフローを実行するよう構成されたakaBotエージェントのグループです。同じプール内のエージェントは、安定した実行のために同一の実行環境である必要があります。

akaBot Centerでエージェントプールを作成するには、**Agent > Agent Pool > Create New** に移動します。

![1771820288041-812.png](/static/img/72eef5_1771820288041-812.png)

*Create Agent Pool*ダイアログでは：

1. エージェントプールのわかりやすい名前を入力します。
2. エージェントが実行するワークフローを選択します。
3. **Token**ドロップダウンで、エージェントプールにメッセージを送信するためのHTTPリクエストに使用する管理トークンを選択します。
   *スケジュール機能のみでトリガーする場合、このステップは任意です。*
4. ワークフローを実行するエージェントを1つ以上選択します。

![1771820332495-836.png](/static/img/6e0ce6_1771820332495-836.png)

作成後、新しいプールがAgent Pool一覧に表示されます。

![1771820349412-497.png](/static/img/1efb90_1771820349412-497.png)

エージェントプールをクリックすると詳細設定を確認できます。

![1771820364096-384.png](/static/img/9420bd_1771820364096-384.png)

**3. スケジュールでエージェントプールをトリガーする**

akaBot Centerのスケジューリング機能を使用してエージェントプールを自動的にトリガーできます。

**Automation > Schedule > Create New** に移動します。

![1771820395129-275.png](/static/img/a4cfc5_1771820395129-275.png)

*Create a new schedule*ダイアログでは：

1. スケジュール名を入力します。
2. 対象のワークフローを選択します（エージェントプールで設定したワークフローと同じであること）。
3. 実行頻度を定義するリカレンス設定を行います。

![1771820414485-470.png](/static/img/4ba9be_1771820414485-470.png)

次に**Forward To**タブへ移動します：

- トリガーしたいAgent Poolを選択します。
- **Save**をクリックして設定を保存します。

![1771820434228-395.png](/static/img/05f330_1771820434228-395.png)

スケジュール作成後、Schedule一覧に表示されます。

![1771820452793-860.png](/static/img/2e595b_1771820452793-860.png)

**実行時の挙動**

スケジュールがトリガーされると：

- 新しいキュー項目が作成されます。
- プール内の利用可能なエージェントがキュー項目をピックアップして実行を開始します。
- 実行中はキュー項目のステータスが**InProgress**になります。
- 利用可能なエージェントがいない場合、キュー項目は**New**のまま待機します。
- 実行後：
  + 正常終了した場合はステータスが**Successful**になります。
  + ビジネス例外が発生した場合は**Failed**になります。

![1771820474256-428.png](/static/img/559350_1771820474256-428.png)

![1771820485476-954.png](/static/img/cc9915_1771820485476-954.png)

**4. サードパーティクライアントからエージェントプールをトリガーする**

![1771820545137-108.png](/static/img/81a915_1771820545137-108.png)

![1771820552772-723.png](/static/img/2502fe_1771820552772-723.png)

リクエストを送信するには以下が必要です：

- **Agent Pool Key**
- 有効な**Managed Token**

例：Postmanを使用してAPIを呼び出すことができます。

![1771820580767-314.png](/static/img/43fead_1771820580767-314.png)

**ワークフローへの入力データの渡し方**

ワークフローに入力データを渡す場合、リクエストボディの**specificData**プロパティにエスケープ済みJSON文字列を含めます。

![1771820599994-165.png](/static/img/1f06b2_1771820599994-165.png)

ワークフローは以下を満たす必要があります：

- **inputData**という名前の引数を**String**型で定義する。
- **Deserialize Json< T >** アクティビティを使用して入力文字列をJSONオブジェクトに変換し処理する。

![1771820619342-196.png](/static/img/f47122_1771820619342-196.png)

**リクエスト状態の照会**

サーバーの応答には**requestId**プロパティが含まれます。後でこの値を使用してリクエストメッセージの状態を照会できます。

![1771820647729-876.png](/static/img/302a99_1771820647729-876.png)

![1771820660862-792.png](/static/img/407911_1771820660862-792.png)
