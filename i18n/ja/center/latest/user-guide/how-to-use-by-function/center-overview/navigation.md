akaBot Centerのナビゲーション構造を理解することは、プラットフォーム上で効率よく作業するために重要です。左側のサイドバーはセクション間を移動する主要な手段であり、業務ユーザーの監視、運用担当者のワークフロー管理、管理者のアクセスや構成管理といったRPAチームの異なる役割に対応する論理的なグループで整理されています。

サイドバーは主に次のセクションに分かれます：**Home**、**Monitoring**（Productivity、Report）、**Automation**（Task、Workflow、Schedule、Package、Trigger）、**Resources**（Agent、Agent Group、Agent Pool）、**Assets**、**Queue**、**Administration**（User Management、Role Management、Organization Unit、Webhook、License、Token）。各セクションは任意に展開/折りたたみ可能で、特定領域で作業するユーザーにとって画面が煩雑にならないよう配慮されています。

メインコンテンツ上部のパンくず（Breadcrumb）は常に現在のアプリ内の位置（例：`Automation > Workflow > Edit Workflow`）を表示します。パンくずの任意のセグメントをクリックするとその階層へ直接移動できます。ヘッダー上部のグローバル検索は、メニューを辿ることなくキーワードでワークフロー、エージェント、パッケージ、ユーザーなどを素早く検索できます。

## ナビゲーション構成

| No | タブ | 説明 |
| --- | --- | --- |
| 1 | Home | akaBot Centerコンポーネントのチャートと統計表示 |
| 2 | Automation | 自動化に関連するすべてのコンポーネントを含みます。 |
| 3 | Resources | エージェント一覧やデプロイ用のエージェントグループを含みます。 |
| 4 | Asset | 開発者が設計プロセスで呼び出し、プロセスで使用できる共有の変数や設定 |
| 5 | Queue | 作業キューはロボットに作業項目を配布するために使用されます。キューの概要、トランザクションステータス、SLA予測など。 |
| 6 | Monitoring | エージェントの活動を監視・追跡し、レポートを表示する場所。 |
| 13 | Administration | *User Management*、Role Management（アクセス権マッピングに基づく認可）、License（ライセンスの有効化/更新/削除と履歴表示）、Webhook（外部システムの購読を許可）など。 |
