# はじめに

**IDP (インテリジェント文書処理) アクティビティ** パッケージは、**akaBot IDP** プラットフォームとシームレスに統合されており、ドキュメントのデジタル化、データ抽出、および分類プロセスを自動化できます。

これらのアクティビティを使用すると、API 資格情報を使用して akaBot IDP サーバーに直接接続し、ロボットが複雑なドキュメント処理ワークフローを処理できるようになります。

これらのアクティビティにより、ロボットは以下のことが可能になります：

* **akaBot IDP への接続**: **IDP Scope** コンテナを介して API キーとサーバー エンドポイントを使用し、安全な接続コンテキストを確立します。
* **ドキュメントのインポートとアップロード**: 自動化されたデータ抽出とデジタル化のために、さまざまなファイル タイプ (請求書、領収書、注文書、PDF、画像など) を IDP サーバーに送信します。
* **ドキュメント ステータスの監視**: アップロードされたドキュメントの処理ステータス (処理中、完了、レビュー待ちなど) をリアルタイムでクエリします。
* **メタデータの取得と更新**: ドキュメントのリストの取得、再処理のためのドキュメントの再ロード、およびドキュメント処理ステータスの更新を行います。
* **構造化データの出力 (エクスポート)**: 後続の自動化タスクで使用するために、処理されたドキュメントから最終的な高精度の抽出データを取得して出力します。

---

## **主な要件**

ワークフローで IDP アクティビティを使用するには：

1. **IDP Scope コンテナ**: 他のすべての IDP アクティビティ (*Import Document*、*Export Document* など) は、接続資格情報を継承するために [IDP Scope](/docs/activities/idp/latest/activities/idp-scope.md) コンテナ内に配置する必要があります。
2. **akaBot IDP アカウント**: サーバーへの接続を許可するには、有効な **サーバー エンドポイント** URL と **API キー** が必要です。
