# Tabs and Alerts Management

> A hands-on guide demonstrating how to manage multiple browser tabs, switch context, and handle JavaScript popups/alerts on a real test website.

This guide provides a comprehensive tutorial for managing advanced browser operations using a real-world testing sandbox: `https://the-internet.herokuapp.com/`. 

You will build workflows that handle browser popups (JS Alerts) and manage multiple tabs/sessions. You will use activities like [Start Browser For Attach](/docs/activities/browser/latest/activities/start-browser-for-attach.md), [Attach Browser](/docs/activities/browser/latest/activities/attach-browser.md), [Browser Scope](/docs/activities/browser/latest/activities/browser-scope.md), [Switch To](/docs/activities/browser/latest/activities/switch-to.md), [Handle Alert](/docs/activities/browser/latest/activities/handle-alert.md), [Go Back](/docs/activities/browser/latest/activities/go-back.md), [Go Forward](/docs/activities/browser/latest/activities/go-forward.md), [Refresh](/docs/activities/browser/latest/activities/refresh.md), and [Quit](/docs/activities/browser/latest/activities/quit.md).

---

## Lesson Overview

* **JavaScript Alert Handling** – Auto-detect and accept browser alert popups.
* **Browser Tabs & Scopes** – Attach akaBot to an existing session, open new tabs, and switch active context between them.
* **Browser History & Quit** – Navigate backwards and forwards in browser history and cleanly close all browser processes.

---

## Section 1: JavaScript Alert Handling

Standard web elements can be automated by clicking them, but browser alerts (modals generated by JavaScript `alert()`, `confirm()`, or `prompt()`) block the DOM. They require specialized handling to accept or dismiss.

### Step-by-Step Instructions

1. Open akaBot Studio and create a new **Process**.
2. Drag an [Open Browser](/docs/activities/browser/latest/activities/open-browser.md) activity inside the main Sequence.
   * In the **Properties** panel under **Input**, set the **Url** field to `"https://the-internet.herokuapp.com/javascript_alerts"`.
   * Set **Browser Type** to `Chrome`.
3. Drag a [Click](/docs/activities/browser/latest/activities/click.md) activity inside the **Do** container of **Open Browser**.
   * Click **Pick target element** and select the button labeled **Click for JS Alert** on the web page.
4. Drag a [Handle Alert](/docs/activities/browser/latest/activities/handle-alert.md) activity immediately below the **Click** activity.
   * Select `ACCEPT` from the **Handle Option** dropdown list in the activity block or in the **Properties** panel.
5. Drag a [Get Text](/docs/activities/browser/latest/activities/get-text.md) activity below **Handle Alert**.
   * Click **Pick target element** and select the result message block below the buttons (which should display *"You successfully clicked an alert"*).
   * Save the result to a variable `alertResult` (using `Ctrl + K` in the **Result** field).
6. Place a **Message Box** activity below the **Get Text** activity to display the content of the `alertResult` variable.
7. Run the process to verify that the robot successfully clicks the button, clicks **OK** on the popup, extracts the result text, and shows it.

## Section 2: Browser Tabs and Scope Management

When automating flows that spawn multiple windows or tabs, akaBot needs to attach to the active browser instance, establish a scope context, and switch active tab focus.

### Step-by-Step Instructions

1. Drag a [Start Browser For Attach](/docs/activities/browser/latest/activities/start-browser-for-attach.md) activity inside your Sequence.
   * In the **Properties** panel, select `Chrome` from the **Browser Type** dropdown list.
2. Inside the **Click to launch browser** container of **Start Browser For Attach**, place an activity that initiates Chrome (such as a [Click](/docs/activities/browser/latest/activities/click.md) activity targeting a desktop browser shortcut, or an [Open Browser](/docs/activities/browser/latest/activities/open-browser.md) activity).
3. Drag an [Attach Browser](/docs/activities/browser/latest/activities/attach-browser.md) activity below the **Start Browser For Attach** container.
   * Select `Chrome` as the browser type.
   * In the **Output Browser** property field of the **Properties** panel, press `Ctrl + K` and declare a new Browser variable named `browserSession`.
4. Drag a [Browser Scope](/docs/activities/browser/latest/activities/browser-scope.md) activity below **Attach Browser**.
   * In the **Browser** property under the **Input** section, type the existing `browserSession` variable directly. Keep the **Output Browser** field empty.
5. Inside the **Do** container of the **Browser Scope** activity:
   * Drag a [Navigate To](/docs/activities/browser/latest/activities/navigate-to.md) activity and set the **Url** to `"https://the-internet.herokuapp.com/windows"`.
   * Drag a [Click](/docs/activities/browser/latest/activities/click.md) activity below it. Click **Pick target element** and select the link text **Click Here**. This link opens a new tab.
   * Drag a [Switch To](/docs/activities/browser/latest/activities/switch-to.md) activity below the Click. In the **Properties** panel, set **Tab Title** to `"New Window"` (or set **Index** to `1`) to shift focus to the new tab.
   * Drag a [Get Text](/docs/activities/browser/latest/activities/get-text.md) activity below **Switch To**. Click **Pick target element** and select the header text `"New Window"` in the active tab. Save the result to a variable `newTabText`.
   * Drag a **Message Box** activity and set its text to `newTabText`.
   * Drag a [Close Tab](/docs/activities/browser/latest/activities/close-tab.md) activity to terminate the active new tab.
   * Drag another [Switch To](/docs/activities/browser/latest/activities/switch-to.md) activity below it. In the **Properties** panel, set **Tab Title** to `"The Internet"` (or set **Index** to `0`) to switch the active session focus back to the original portal home tab.
   * Drag a [Go Back](/docs/activities/browser/latest/activities/go-back.md) activity to return to the home page of the website.
   * Drag a [Go Forward](/docs/activities/browser/latest/activities/go-forward.md) activity to navigate forward to the windows test page.
   * Drag a [Refresh](/docs/activities/browser/latest/activities/refresh.md) activity to reload the active page.
6. Drag a [Quit](/docs/activities/browser/latest/activities/quit.md) activity at the very bottom of the main workflow sequence (outside the **Browser Scope** container).
7. Run the process. akaBot attaches to the active browser instance, navigates to the multiple windows testing page, clicks to spawn a tab, switches to and reads the tab content, closes the tab, returns to the original window, navigates back and forward, and quits the entire browser application.
