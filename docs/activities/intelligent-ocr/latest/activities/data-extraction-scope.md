# Data Extraction Scope

RCA.Activities.OCR.DataExtractionScope

## **Description**

The **Data Extraction Scope** activity provides a execution and configuration container for nested extractor activities (such as the [Form Extractor](/docs/activities/intelligent-ocr/latest/extractors/form-extractor.md) or [Regular Expression Extractor](/docs/activities/intelligent-ocr/latest/extractors/regex-based-extractor.md)). It coordinates the extraction process according to the document types and fields defined in your taxonomy.

The activity takes the digitized representation of the document (its Document Text and Document Object Model) and a specific Document Type ID, runs the configured extractors inside its body, and stores the results in an `ExtractionResult` variable. This output variable can then be passed into the [Export Extraction Results](/docs/activities/intelligent-ocr/latest/activities/export-extraction-results.md) activity to retrieve the data in a tabular format.

## **In the body of the activity**

* **Drop Extractor Here** - The central container block where you must drag and drop one or more extractor activities (such as *Form Extractor*) to define the extraction logic.

---

## **Properties**

### **Input**

* **Document Object Model (Document)**\* - The Document Object Model (DOM) of the file, typically stored in a `Document` variable generated by a preceding [Digitize Document](/docs/activities/intelligent-ocr/latest/activities/digitize-document.md) activity.
* **Document Text (String)**\* - The raw text content of the document itself, typically stored in a `String` variable generated by a preceding [Digitize Document](/docs/activities/intelligent-ocr/latest/activities/digitize-document.md) activity.
* **Document Type Id (String)**\* - The unique Document Type ID from your taxonomy configuration that identifies the document type to process (e.g., `"Invoices.Inbound.VendorInvoice"`).

### **Misc**

* **Display Name (String)** - The name of this activity in the workflow designer panel.  
  E.g: `[755350182] Data Extraction Scope`.
* **Public (Checkbox)** - If selected, the values of variables and arguments used in this activity are logged at the Verbose level. Consider data security requirements before checking this option.

### **Output**

* **Extraction Result (ExtractionResult)** - An akaBot variable of type `ExtractionResult` that stores the processed and extracted data.
