# Reset Form

RCA.Activities.FormBuilder.ResetForm

## **Description**
Clears all input data entered by the user in the form window, restoring all input elements (text boxes, dropdowns, checkboxes) back to their default state or values defined in the JSON schema. This is useful for implementing a "Clear Form" action or preparing the form layout for a new entry cycle without closing the window.

![resert-form](/static/img/resert-form.png)

---

## **In the body of the activity**
* **Form Id** - The ID of the active form window you want to clear. This must be the output variable obtained from the parent [Display Form](/docs/activities/form-builder/latest/activities/display-form.md) activity.

## **Step-by-Step Usage**
The **Reset Form** activity operates on a live form instance, so it must be executed inside the **Do** block container.

For example, to implement a custom clear action:
1. **Place the Activity**: Drag the **Reset Form** activity inside the **Do** block of [Display Form](/docs/activities/form-builder/latest/activities/display-form.md).
2. **Listen to Button Clicks**: Use an **If** activity inside the **Do** block to check if the user clicked the reset button. Set the condition to check the **Selected Key** output variable of `Display Form` (e.g. `selectedKey = "btnClear"`).
3. **Execute Reset**: Inside the **Then** branch of the **If** activity, call **Reset Form** and bind the **Form Id** property to your form's ID variable (e.g. `formId`). When the user clicks the "Clear" button, all form fields will immediately clear and reset to their default values.

## **Self-Service Troubleshooting**

### **Design-Time Validation**
* **Symptom**: Studio displays a validation warning that `Form Id` is required.
  * **Cause**: The `Form Id` field is empty or a non-integer variable is passed.
  * **Solution**: Create an `Int32` variable (e.g. `myFormId`), bind it to the output **Form Id** of `Display Form`, and pass it to this activity.

### **Run-Time Errors**
* **Symptom**: `ArgumentException: Form ID does not exist` at runtime.
  * **Cause**: The form window has already been closed (either by the user manually or via `Timeout MS`).
  * **Solution**: Ensure this activity is executed within the boundaries of the **Do** block execution loop.
