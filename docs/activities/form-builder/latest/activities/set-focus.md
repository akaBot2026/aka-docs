# Set Focus

`RCA.Activities.FormBuilder.SetFocus`

## **Description**
Sets visual focus (places the active input cursor) onto a specific input component on an active form. This is used to guide the end-user's attention to the next required field, or to highlight a field that failed validation check during dynamic form processing.

![set-focus](/static/img/set-focus.png)

---

## **In the body of the activity**
* **Key Name** - The unique key name of the form element on which you want to place focus. This value must match the component's Property Name (Field Key) from the Form Builder layout, and must be enclosed in double quotes (e.g., `"txtAge"`).

## **Step-by-Step Usage**
The **Set Focus** activity operates on a live form instance, so it must be executed inside the **Do** block container.

For example, to guide a user through a sequential input:
1. In the **Do** block, the robot uses [Get Element Value](/docs/activities/form-builder/latest/activities/get-element-value.md) to inspect the value of the email field (`"txtEmail"`).
2. If the user moves to the next field but left the email blank or invalid, the robot displays an inline warning and calls **Set Focus** on `"txtEmail"` to force the cursor back to that field until it is corrected.

## **Self-Service Troubleshooting**

### **Design-Time Validation**
* **Symptom**: Studio displays a warning that `Key Name` or `Form Id` is required.
  * **Cause**: One of these properties is empty or the `Key Name` string lacks quotes.
  * **Solution**: Ensure `Key Name` is formatted as a string (e.g. `"txtFirstName"`) and `Form Id` is mapped to your `Form Id` variable.

### **Run-Time Errors**
* **Symptom**: The cursor does not appear on the element, but no error is thrown.
  * **Cause**:
    1. The target component is currently hidden or disabled using the [Disable](/docs/activities/form-builder/latest/activities/disable.md) activity. Hidden or disabled components cannot receive focus.
    2. Another UI window has captured the global screen focus.
  * **Solution**: Ensure the component is active (call [Enable](/docs/activities/form-builder/latest/activities/enable.md) first if it was previously disabled) before calling **Set Focus**.
* **Symptom**: `KeyNotFoundException: Component key 'xyz' not found`.
  * **Cause**: The key is misspelled or does not exist in the schema.
  * **Solution**: Check the component's Field Key configuration in the Form Builder layout designer.
