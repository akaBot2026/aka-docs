# Disable

RCA.Activities.FormBuilder.Disable

## **Description**
Disables a specific user interface component (e.g. text box, dropdown list, button) on an active form. When disabled, the component becomes grayed-out and the end-user cannot enter text, select options, or click on it. This is useful for building dynamic form behaviors where elements are only editable when certain criteria are met.

![disable](/static/img/disable.png)

(\*For mandatory)

## **In the body of the activity**
* **Key Name** - The unique key name of the form element you want to disable. This value must match the component's Property Name (Field Key) from the Form Builder layout, and must be enclosed in double quotes (e.g., `"btnSubmit"`).

## **Step-by-Step Usage**
The **Disable** activity must run inside the **Do** block of [Display Form](/docs/activities/form-builder/latest/activities/display-form.md) because it operates on a live form instance.

For example, to disable the submit button until the user types their name:
1. Drag a **Get Element Value** activity inside the **Do** block of `Display Form` to read the Full Name input field (`"txtFullName"`).
2. Place an **If** activity: check if the retrieved name is empty.
3. If it is empty, call **Disable** for the submit button key (`"btnSubmit"`).
4. If it is not empty, call [Enable](/docs/activities/form-builder/latest/activities/enable.md) for the `"btnSubmit"` key.

## **Self-Service Troubleshooting**

### **Design-Time Validation**
* **Symptom**: Studio displays a warning that `Key Name` or `Form Id` is required.
  * **Cause**: One of these properties is empty or the `Key Name` string lacks quotes.
  * **Solution**: Ensure `Key Name` is formatted as a string (e.g. `"btnSubmit"`) and `Form Id` is mapped to your `Form Id` variable.

### **Run-Time Errors**
* **Symptom**: `KeyNotFoundException: Component key 'xyz' not found` at runtime.
  * **Cause**: The key passed in **Key Name** (e.g. `"xyz"`) does not match any Field Key (Property Name) inside the JSON schema file.
  * **Solution**: Open the JSON schema file in Form Builder, verify the component's **Property Name** under the **Field Key** tab, and update the **Key Name** to match it exactly (case-sensitive).
* **Symptom**: `ArgumentException: Form ID does not exist`.
  * **Cause**: The form is already closed.
  * **Solution**: Ensure this activity is executed within the boundaries of the **Do** block execution loop.
